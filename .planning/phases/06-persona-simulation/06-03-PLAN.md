# Plan 06-03: Persona Editor Panel + E2E Validation

## Objective
Persona editor UI in agent detail, API routes for persona operations, and E2E validation.

## Context
- Requirements: PRSA-01, PRSA-02, PRSA-03, PRSA-08
- Existing: agent-detail-tabs.tsx has OverviewTab, SoulTab, MemoryTab, TasksTab, ActivityTab, ConfigTab
- Existing: persona-engine exports applyPreset(), getPresetNames(), BigFive interface
- PAD + biases + trust from 06-01 and 06-02

## Tasks

### Task 1: Persona API routes
Create `src/app/api/agents/[id]/persona/route.ts`:

**GET** — returns persona config, PAD state, active biases, pairwise trust network
**PUT** — update OCEAN traits, apply preset, update PAD state

### Task 2: Persona editor tab
Add PersonaTab to `src/components/panels/agent-detail-tabs.tsx`:

**OCEAN Traits editor:**
- 5 sliders (range 0-1, step 0.1) for each Big Five dimension
- Labels: Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism
- Visual bar for each trait value

**PAD Emotional State:**
- 3 sliders (range -1 to 1, step 0.1): Pleasure, Arousal, Dominance
- Current emotion label (from padToEmotionLabel)
- "Reset to baseline" button

**Preset selector:**
- Dropdown with getPresetNames()
- "Apply Preset" button that overwrites current OCEAN traits

**Active Biases display:**
- Read-only list of currently active biases based on traits
- Each bias shows name + description

**Trust Network:**
- List of other agents with trust score bars
- Read-only display

### Task 3: Register PersonaTab in agent detail
Add `'persona'` tab to the agent detail tab system (wherever tabs are defined).

### Task 4: E2E specs + full suite validation
Create `tests/persona.spec.ts`:
- GET /api/agents/[id]/persona returns defaults
- PUT OCEAN traits and verify persistence
- Apply preset and verify traits updated
- PUT PAD state and verify
- Verify active biases match trait thresholds

Run full test suite: typecheck + unit + E2E.
