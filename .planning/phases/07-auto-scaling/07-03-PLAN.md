# Plan 07-03: Scaling Monitor Panel + E2E Validation

## Objective
Scaling dashboard panel showing pool size, idle/busy counts, pending tasks, event history, and policy management. Plus E2E validation of the full scaling lifecycle.

## Context
- Requirements: SCAL-09, SCAL-10
- Existing: scaling-engine.ts + API routes from 07-01 and 07-02
- Existing: workflow-panel.tsx as panel template pattern
- Existing: /api/workload endpoint with queue depth, agent counts, recommendations

## Tasks

### Task 1: Scaling monitor panel
Create `src/components/panels/scaling-panel.tsx`:

**Overview section:**
- Current pool size (total agents / global cap)
- Agent status breakdown: busy, idle, offline, error (color-coded)
- Queue depth (pending tasks) with threshold indicator
- Last evaluation timestamp

**Policies tab:**
- List of scaling policies with enabled toggle
- Create policy form (name, min/max agents, thresholds, cooldown, auto_approve, template)
- Edit/delete policy

**Events tab:**
- Event history table (type, status, agent, reason, created_at, resolved_at)
- Status badges: pending=yellow, approved=green, rejected=red, completed=blue
- Approve/Reject buttons for pending events
- Metrics snapshot expandable detail

**Manual evaluation:**
- "Evaluate Now" button per policy
- Shows result (event created or "no action needed")

### Task 2: Register scaling panel
Add `case 'scaling'` to ContentRouter in `src/app/[[...panel]]/page.tsx`.

### Task 3: E2E specs
Create `tests/scaling.spec.ts`:
- Create scaling policy (POST /api/scaling/policies)
- List policies (GET /api/scaling/policies)
- Update policy (PUT /api/scaling/policies/[id])
- Delete policy (DELETE /api/scaling/policies/[id])
- Evaluate scaling with no pressure (returns null/no event)
- Create tasks to exceed threshold, evaluate, verify scale_up event created
- Approve scale-up event, verify agent created
- Reject scale-down event, verify status updated
- Global agent cap enforcement
- Auto-approve mode creates and executes immediately
- 401 without API key

Run full test suite: typecheck + unit + E2E.
