# Plan 07-02: Scaling API Routes + Approval Gate

## Objective
API routes for scaling policy CRUD, manual evaluation trigger, approval/rejection of pending scale events, and event history.

## Context
- Requirements: SCAL-04, SCAL-05, SCAL-06, SCAL-08
- Existing: scaling-engine.ts from 07-01 with evaluateScaling, executeScaleUp, executeScaleDown
- Existing: POST /api/agents creates agents with template config
- Existing: requireRole auth pattern, workspaceId filtering
- Pattern: debates API routes as reference for CRUD + state transitions

## Tasks

### Task 1: Scaling policies CRUD
Create `src/app/api/scaling/policies/route.ts`:

**GET** — list all policies (with optional `?enabled=1` filter)
**POST** — create policy (name, min_agents, max_agents, scale_up_threshold, scale_down_threshold, cooldown_seconds, idle_timeout_seconds, auto_approve, agent_template)

Create `src/app/api/scaling/policies/[id]/route.ts`:

**GET** — single policy with recent events
**PUT** — update policy fields
**DELETE** — delete policy (cascade events)

### Task 2: Scaling evaluation + approval routes
Create `src/app/api/scaling/evaluate/route.ts`:

**POST** — trigger evaluation for a policy (body: { policyId }). Returns event or null.

Create `src/app/api/scaling/events/route.ts`:

**GET** — list scaling events (optional filters: ?status=pending, ?policy_id=X, ?limit=50)

Create `src/app/api/scaling/events/[id]/route.ts`:

**PUT** — approve or reject pending event:
  - `{ action: 'approve' }` → if scale_up: call executeScaleUp; if scale_down: call executeScaleDown
  - `{ action: 'reject' }` → set status to 'rejected', set resolved_at
  - If policy.auto_approve is true AND event was just created, auto-execute immediately

### Task 3: Unit tests for API routes
Create `src/lib/__tests__/scaling-api.test.ts`:
- Policy CRUD (create, list, get, update, delete)
- Evaluate returns event when threshold exceeded
- Evaluate returns null within cooldown
- Approve scale-up creates agent
- Approve scale-down retires agent
- Reject event sets status
- Auto-approve executes immediately
