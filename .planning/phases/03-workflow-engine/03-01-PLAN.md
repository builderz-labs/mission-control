# Plan 03-01: Workflow Runs Table + Phase State Machine

## Objective
Add a `workflow_runs` table to track run metadata and phase progression, plus extend `workflow_templates` with phase definitions.

## Context
- Requirements: WKFL-01, WKFL-03, WKFL-04, WKFL-07
- Existing: sop_messages + sop_role_state tables, workflow_templates table, sop-engine.ts
- The existing engine uses message-driven state but lacks explicit phase tracking. We need a workflow_runs table and a workflow_phases table.

## Tasks

### Task 1: Migration for workflow_phases table
Add `phase_045_workflow_phases` to phase-migrations.ts:

```sql
CREATE TABLE IF NOT EXISTS workflow_phases (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  template_id INTEGER NOT NULL REFERENCES workflow_templates(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  phase_order INTEGER NOT NULL,
  agent_role TEXT,
  input_schema TEXT,   -- JSON Zod schema definition
  output_schema TEXT,  -- JSON Zod schema definition
  requires_approval INTEGER NOT NULL DEFAULT 0,
  description TEXT,
  UNIQUE(template_id, phase_order)
);
CREATE INDEX IF NOT EXISTS idx_wf_phases_template ON workflow_phases(template_id);
```

### Task 2: Migration for workflow_runs table
Add `phase_046_workflow_runs` to phase-migrations.ts:

```sql
CREATE TABLE IF NOT EXISTS workflow_runs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  template_id INTEGER NOT NULL REFERENCES workflow_templates(id) ON DELETE CASCADE,
  status TEXT NOT NULL DEFAULT 'pending' CHECK(status IN ('pending','running','paused','completed','failed')),
  current_phase_id INTEGER REFERENCES workflow_phases(id),
  input_data TEXT,  -- JSON initial input
  started_at INTEGER,
  completed_at INTEGER,
  workspace_id INTEGER NOT NULL DEFAULT 1,
  created_by TEXT NOT NULL DEFAULT 'system',
  created_at INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE INDEX IF NOT EXISTS idx_wf_runs_template ON workflow_runs(template_id);
CREATE INDEX IF NOT EXISTS idx_wf_runs_status ON workflow_runs(status);
CREATE INDEX IF NOT EXISTS idx_wf_runs_workspace ON workflow_runs(workspace_id);
```

### Task 3: Migration for workflow_phase_runs table
Add `phase_047_workflow_phase_runs` to phase-migrations.ts:

```sql
CREATE TABLE IF NOT EXISTS workflow_phase_runs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  run_id INTEGER NOT NULL REFERENCES workflow_runs(id) ON DELETE CASCADE,
  phase_id INTEGER NOT NULL REFERENCES workflow_phases(id) ON DELETE CASCADE,
  status TEXT NOT NULL DEFAULT 'pending' CHECK(status IN ('pending','running','paused','completed','failed','rejected')),
  input_artifact TEXT,  -- JSON
  output_artifact TEXT, -- JSON
  validation_error TEXT,
  approved_by TEXT,
  approved_at INTEGER,
  started_at INTEGER,
  completed_at INTEGER,
  UNIQUE(run_id, phase_id)
);
CREATE INDEX IF NOT EXISTS idx_wf_phase_runs_run ON workflow_phase_runs(run_id);
```

### Task 4: Workflow phase engine
Create `src/lib/workflow-engine.ts`:

Core functions:
- `createWorkflowRun(templateId, inputData, createdBy, workspaceId)` → run ID
- `advanceWorkflow(runId)` — check current phase status, validate output, advance to next
- `completePhase(runId, phaseId, outputArtifact)` — validate against schema, mark complete
- `approvePhase(runId, phaseId, approvedBy)` — for approval gates
- `rejectPhase(runId, phaseId, reason)` — reject and pause workflow
- `getWorkflowRunStatus(runId)` — full run + phase states

State machine: pending → running → (per phase: pending → running → [paused if approval needed] → completed/rejected) → completed/failed

### Task 5: Unit tests
Create `src/lib/__tests__/workflow-engine.test.ts` — test state transitions, phase advancement, validation, approval gates.
