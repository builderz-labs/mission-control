# Plan 03-02: Workflow API Routes

## Objective
CRUD API routes for workflow templates with phases, workflow runs, and phase transitions.

## Context
- Requirements: WKFL-01, WKFL-02, WKFL-03, WKFL-05, WKFL-08, WKFL-09
- Existing: /api/workflows for template CRUD, /api/workflows/sop/start for SOP runs
- New routes build on workflow-engine.ts from 03-01

## Tasks

### Task 1: Extend template CRUD with phases
Modify `src/app/api/workflows/route.ts`:
- POST: Accept optional `phases` array with { name, phase_order, agent_role, input_schema, output_schema, requires_approval, description }
- GET: Include phases in template response (JOIN workflow_phases)
- PUT: Allow updating phases (delete+re-insert pattern within transaction)

### Task 2: Template phases sub-route
Create `src/app/api/workflows/[id]/phases/route.ts`:
- GET: List phases for a template
- POST: Add phase to template
- PUT: Reorder/update phases

### Task 3: Workflow runs routes
Create `src/app/api/workflows/runs/route.ts`:
- GET: List runs with status filter, workspace scoped
- POST: Start a new workflow run from template (delegates to createWorkflowRun)

Create `src/app/api/workflows/runs/[id]/route.ts`:
- GET: Full run status with phase states

### Task 4: Phase transition routes
Create `src/app/api/workflows/runs/[id]/advance/route.ts`:
- POST: Submit phase output and advance (delegates to completePhase + advanceWorkflow)

Create `src/app/api/workflows/runs/[id]/approve/route.ts`:
- POST: Approve paused phase (delegates to approvePhase)

Create `src/app/api/workflows/runs/[id]/reject/route.ts`:
- POST: Reject phase with reason

### Task 5: EventBus broadcasts
Broadcast on every transition:
- `workflow.run.started` on run creation
- `workflow.phase.transition` on phase advancement
- `workflow.phase.approval_required` when hitting approval gate
- `workflow.run.completed` on final phase completion
