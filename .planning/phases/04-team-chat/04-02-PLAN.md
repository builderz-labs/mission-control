# Plan 04-02: Teams + Chat API Routes

## Objective
CRUD API routes for teams + team membership, and wire mention routing into chat message POST.

## Context
- Requirements: CHAT-01, CHAT-04, CHAT-05, CHAT-07, CHAT-08, CHAT-09
- Existing: POST /api/chat/messages already handles message creation + gateway forwarding
- Existing: /api/mentions returns autocomplete targets
- New: mention-router.ts from 04-01

## Tasks

### Task 1: Teams CRUD routes
Create `src/app/api/teams/route.ts`:
- GET: List teams with member count (workspace scoped)
- POST: Create team with name, description
- PUT: Update team
- DELETE: Delete team (cascade members)

### Task 2: Team members routes
Create `src/app/api/teams/[id]/members/route.ts`:
- GET: List members of team (returns agents with roles)
- POST: Add agent to team
- DELETE: Remove agent from team

### Task 3: Wire mention routing into chat POST
Modify `src/app/api/chat/messages/route.ts`:
- After message insert, call `routeMentions(db, messageId, content, conversationId, workspaceId)`
- Include routing result in response: `{ message, mentionRouting: { routed, unresolved } }`

### Task 4: Update /api/mentions for teams
Modify `src/app/api/mentions/route.ts`:
- Include teams in autocomplete results (type: 'team', handle: 'team:name')
- Filter by query across agents, users, AND teams

### Task 5: Unit tests for teams API
Create `src/lib/__tests__/teams-api.test.ts`:
- Team name validation
- Member uniqueness constraint
- Workspace scoping
