# Plan 08-01: Cross-System SSE Wiring

## Objective
Wire SSE event listeners so workflow, debate, scaling, and persona events propagate across UI panels and the spatial canvas in real-time.

## Context
- Requirements: QUAL-05 (cross-system SSE events)
- EventBus broadcasts 41 events but only core (task/agent/chat) and spatial events are consumed by UI
- `useServerEvents()` ignores workflow/debate/scaling/persona events
- `useSpatialSSE()` ignores workflow/scaling/persona events
- Individual panels (workflow, debate, scaling) use polling, not SSE

## Tasks

### Task 1: Extend useServerEvents for new systems
In `src/lib/use-server-events.ts`, add SSE event handlers for:
- `workflow.phase.approval_required` → dispatch notification to notification store
- `scaling.hire.approved` → refresh agent list
- `scaling.retire.initiated` → refresh agent list
- `debate.concluded` → no-op (just log for now, panels handle their own)

### Task 2: Extend useSpatialSSE for cross-system events
In `src/hooks/use-spatial-sse.ts`, add handlers for:
- `scaling.hire.approved` → trigger agent refetch (node will appear via `agent.created`)
- `workflow.phase.transition` → no direct spatial effect (agents already tracked)

### Task 3: Add SSE hooks to panels
- **Workflow panel:** Add `useEffect` listening to EventSource for `workflow.*` events to auto-refresh active runs
- **Debate panel:** Add `useEffect` listening for `debate.*` events to auto-refresh active debate detail
- **Scaling panel:** Add `useEffect` listening for `scaling.*` events to auto-refresh metrics and events

### Task 4: Unit tests for SSE integration
Create `src/lib/__tests__/sse-integration.test.ts`:
- Test that useServerEvents handler map includes workflow/scaling events
- Test that event type dispatching is correct
