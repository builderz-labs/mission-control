# Plan 05-01: Debate Schema + Engine

## Objective
Create dedicated debate tables, structured round engine (propose/critique/rebut/vote), token budget enforcement.

## Context
- Requirements: DEBT-01, DEBT-02, DEBT-03, DEBT-04, DEBT-05, DEBT-07
- Existing: conversation-engine.ts with startDebate() (basic round-robin), conversation_state table, 6 EventBus debate events
- Pattern: Follow workflow-engine.ts state machine pattern with writeTransaction()

## Tasks

### Task 1: Add debate migrations
Add to `src/lib/phase-migrations.ts`:
- `phase_050_debates` — debates table:
  - id, topic, status (pending/propose/critique/rebut/vote/concluded/budget_exhausted), current_round, max_rounds, token_budget, tokens_used, outcome, vote_accept, vote_reject, workspace_id, created_by, created_at, concluded_at
- `phase_051_debate_arguments` — debate_arguments table:
  - id, debate_id, agent_id, agent_name, round_number, phase (propose/critique/rebut), content, confidence, tokens_used, created_at
- `phase_052_debate_votes` — debate_votes table:
  - id, debate_id, agent_id, agent_name, vote (accept/reject), reason, created_at, UNIQUE(debate_id, agent_id)
- `phase_053_debate_participants` — debate_participants table:
  - debate_id, agent_id, agent_name, joined_at, PRIMARY KEY(debate_id, agent_id)

### Task 2: Create debate-engine.ts
Create `src/lib/debate-engine.ts` with structured round state machine:

```
createDebate(db, topic, participantIds[], maxRounds, tokenBudget, createdBy, workspaceId)
  → Creates debate + participants, broadcasts debate.created

submitArgument(db, debateId, agentId, content, confidence)
  → Validates phase, stores argument, broadcasts debate.argument.submitted
  → Tracks token usage, returns budget status

advanceDebatePhase(db, debateId)
  → propose → critique → rebut → vote (per round), or propose (next round)
  → Broadcasts debate.round.started / debate.round.completed

castVote(db, debateId, agentId, vote: 'accept'|'reject', reason?)
  → Stores vote, checks if all participants voted
  → If all voted: tallies, checks consensus threshold (>50%), concludes or advances

getDebateStatus(db, debateId)
  → Returns full debate state with arguments, votes, participants

concludeDebate(db, debateId, outcome)
  → Sets final outcome, broadcasts debate.concluded
```

Token budget: track tokens_used per argument (estimate from content.length / 4). If tokens_used >= token_budget before any LLM call, set status = 'budget_exhausted'.

### Task 3: Unit tests for debate-engine
Create `src/lib/__tests__/debate-engine.test.ts`:
- createDebate (valid, no agents, token budget)
- submitArgument (valid, wrong phase, budget exceeded)
- advanceDebatePhase (propose→critique→rebut→vote, round transitions)
- castVote (valid, duplicate, all voted → concludes)
- getDebateStatus (full state)
