# Plan 05-02: Debate API Routes

## Objective
CRUD + lifecycle API routes for debates.

## Context
- Requirements: DEBT-01, DEBT-02, DEBT-04, DEBT-06, DEBT-09
- Existing: /api/debates/start route (basic, delegates to conversation-engine startDebate)
- Pattern: Follow teams and workflow routes with requireRole, validateBody, getDatabase

## Tasks

### Task 1: Create debate routes
Create/update API routes:

**`src/app/api/debates/route.ts`** (new):
- GET: List debates with pagination, status filter, workspace scoped
- POST: Create debate (topic, participantIds, maxRounds, tokenBudget)
  - Validates participants exist, creates via debate-engine.createDebate()
- DELETE: Delete debate (admin only)

**`src/app/api/debates/[id]/route.ts`** (new):
- GET: Full debate status (arguments, votes, participants, current round/phase)

**`src/app/api/debates/[id]/arguments/route.ts`** (new):
- GET: List arguments for debate (optional round/phase filter)
- POST: Submit argument (agentId, content, confidence)

**`src/app/api/debates/[id]/advance/route.ts`** (new):
- POST: Advance debate phase (operator)

**`src/app/api/debates/[id]/vote/route.ts`** (new):
- POST: Cast vote (agentId, vote: accept/reject, reason)

**`src/app/api/debates/[id]/results/route.ts`** (new):
- GET: Debate results (outcome, vote tally, argument summary per round)

### Task 2: Update existing /api/debates/start
Modify `src/app/api/debates/start/route.ts`:
- Keep for backward compat but delegate to new createDebate from debate-engine
- Return debate object with ID instead of running full conversation

### Task 3: Debate API unit tests
Create `src/lib/__tests__/debate-api.test.ts`:
- Debate CRUD validation
- Argument submission validation
- Vote casting validation
- Phase advancement validation
