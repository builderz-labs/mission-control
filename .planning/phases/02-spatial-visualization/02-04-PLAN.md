# Plan 02-04: Performance + 50-Node Stress Test

## Objective
Ensure the spatial canvas handles 50+ agents without frame drops below 30fps (SPAT-10), and optimize rendering.

## Context
- Requirements: SPAT-10 (50+ nodes at 30fps)
- React Flow performance: Use `nodesDraggable`, `nodesConnectable=false`, `minZoom/maxZoom`, `onlyRenderVisibleElements` (default in RF 12)
- Key: Avoid re-rendering all nodes when one node's status updates

## Tasks

### Task 1: React.memo on custom nodes
Wrap `AgentNodeComponent` and `TeamGroupNode` with `React.memo()` and a custom comparator that only re-renders when `data.status`, `data.label`, or `data.role` change. This prevents 50 node re-renders when only 1 node updates.

### Task 2: Batch SSE updates
In `use-spatial-sse.ts`, batch rapid-fire events using `requestAnimationFrame` debouncing. When 10 agents update status in quick succession, apply all changes in a single store update rather than 10 separate ones.

### Task 3: Performance props on ReactFlow
In SpatialCanvasPanel, set:
```typescript
<ReactFlow
  nodes={nodes}
  edges={edges}
  nodeTypes={agentNodeTypes}
  edgeTypes={agentEdgeTypes}
  nodesConnectable={false}       // Users don't create edges via drag
  minZoom={0.1}
  maxZoom={2}
  fitView
  proOptions={{ hideAttribution: true }}
>
```

### Task 4: Stress test (unit test)
Create `src/components/spatial/__tests__/canvas-perf.test.ts`:
- Generate 50 mock agent nodes + 80 edges
- Verify dagre layout completes in < 100ms
- Verify canvas store handles 50 `addNode` calls without error
- Verify `updateNodeData` on 1 of 50 nodes doesn't trigger full re-layout

### Task 5: E2E performance spec
Create `tests/spatial-canvas.spec.ts`:
- Navigate to `/spatial`
- Seed 50 agents via API
- Verify canvas renders (check for `.react-flow__node` elements)
- Verify node count matches expected

## Verification
- [ ] Custom nodes use React.memo with shallow data compare
- [ ] SSE updates batched per animation frame
- [ ] 50-node dagre layout < 100ms
- [ ] Canvas renders 50 nodes in E2E test
- [ ] No frame drops visible (qualitative — E2E can't measure fps directly)
- [ ] Tests pass
