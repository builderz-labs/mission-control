# Plan 02-01: Database Schema + API Routes for Agent Relationships

## Objective
Create the `agent_relationships` table and CRUD API routes so the spatial canvas has data to render edges between agents.

## Context
- Requirements: SPAT-02 (relationship edges), SPAT-06 (SSE-driven updates)
- Existing: `agents` table in schema.sql, CRUD at `/api/agents/`, phase-migrations.ts pattern
- Convention: `registerMigrations()` in phase-migrations.ts, workspace_id scoping, requireRole() + getDatabase() pattern

## Tasks

### Task 1: Migration for agent_relationships table
Add migration `phase_043_agent_relationships` to `src/lib/phase-migrations.ts`:

```sql
CREATE TABLE IF NOT EXISTS agent_relationships (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source_agent_id INTEGER NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
  target_agent_id INTEGER NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
  type TEXT NOT NULL CHECK(type IN ('delegation','communication','supervision')),
  metadata TEXT,  -- JSON for edge-specific config
  workspace_id INTEGER NOT NULL DEFAULT 1,
  created_at INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at INTEGER NOT NULL DEFAULT (unixepoch()),
  UNIQUE(source_agent_id, target_agent_id, type)
);
CREATE INDEX IF NOT EXISTS idx_agent_rel_source ON agent_relationships(source_agent_id);
CREATE INDEX IF NOT EXISTS idx_agent_rel_target ON agent_relationships(target_agent_id);
CREATE INDEX IF NOT EXISTS idx_agent_rel_workspace ON agent_relationships(workspace_id);
```

### Task 2: Migration for spatial_positions table
Add migration `phase_044_spatial_positions` to `src/lib/phase-migrations.ts`:

```sql
CREATE TABLE IF NOT EXISTS spatial_positions (
  agent_id INTEGER PRIMARY KEY REFERENCES agents(id) ON DELETE CASCADE,
  x REAL NOT NULL DEFAULT 0,
  y REAL NOT NULL DEFAULT 0,
  workspace_id INTEGER NOT NULL DEFAULT 1,
  updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);
```

This persists user-dragged positions server-side (SPAT-04). The canvas store's localStorage is a client cache; this is the source of truth.

### Task 3: API route — GET/POST /api/spatial/relationships
Create `src/app/api/spatial/relationships/route.ts`:

- **GET**: List all relationships for workspace. Join agents table for source/target names.
  - Query params: `type` (optional filter), `agent_id` (optional filter for specific agent)
  - Returns: `{ relationships: RelationshipRow[] }`
- **POST**: Create a relationship.
  - Body: `{ source_agent_id, target_agent_id, type, metadata? }`
  - Validates both agents exist in workspace
  - Broadcasts `spatial.node.added` event (reuse — or add `spatial.edge.added`)
  - Returns: `{ relationship: RelationshipRow }`, status 201

### Task 4: API route — DELETE /api/spatial/relationships/[id]
Create `src/app/api/spatial/relationships/[id]/route.ts`:

- **DELETE**: Remove relationship by ID
  - Validates relationship exists and belongs to workspace
  - Broadcasts event
  - Returns: `{ success: true }`

### Task 5: API route — GET/PUT /api/spatial/positions
Create `src/app/api/spatial/positions/route.ts`:

- **GET**: Fetch all saved positions for workspace
  - Returns: `{ positions: { agent_id: number, x: number, y: number }[] }`
- **PUT**: Batch-upsert positions (called after drag)
  - Body: `{ positions: { agent_id: number, x: number, y: number }[] }`
  - Uses `INSERT OR REPLACE` for each position
  - Returns: `{ success: true, count: number }`

### Task 6: Add EventBus edge events
Add to EventDataMap in `src/lib/event-bus.ts`:
```typescript
'spatial.edge.added': { edgeId: string; sourceAgentId: number; targetAgentId: number; type: string }
'spatial.edge.removed': { edgeId: string }
'spatial.positions.updated': { count: number }
```

### Task 7: Unit tests
Create `src/lib/__tests__/spatial-api.test.ts` — test relationship validation (both agents exist, type constraint), position upsert logic.
